一、模型原理必须脱稿讲清楚
---------------------------

1.1 ACT
-------
[ ] 画一张 ACT 架构图：
    image encoder (ResNet) + state encoder + transformer encoder + transformer decoder + action head

[ ] 背熟 3 个关键参数：
    - chunk_size=50：一次性预测 50 步 action
    - n_obs_steps=1：用 1 帧历史观测
    - temporal_ensemble_coeff：推理时 ensemble 的权重衰减系数

[ ] 分清训练和推理：
    - 训练：对预测的 50 步 action 和 ground truth 的 50 步 action 逐帧算 L1 loss，再平均
    - 推理：当前时刻预测一个 chunk，和历史 chunk 按权重平均（temporal ensemble），执行前 n_action_steps 步

[ ] 准备一句话总结：
    "ACT 用 transformer 预测 action chunk，靠 temporal ensemble 平滑执行，
     但 MSE/L1 损失会把多模态动作平均成四不像。"

面试话术示例：
    "ACT 的核心是 action chunking + temporal ensemble。训练时模型输出 50 步
     action，和示范数据的 50 步算 L1 loss；推理时为了平滑，会把当前 chunk 和
     过去几个 chunk 加权平均，只执行前 20 步。这种设计让动作连贯，但面对多
     模态分布时，MSE 会把多个正确轨迹平均成一个不可行的轨迹，这就是它在
     stack_bowls 上抽搐的原因。"


1.2 Diffusion Policy
--------------------
[ ] 画一张 Diffusion Policy 架构图：
    image encoder + state encoder + FiLM conditioning + UNet 1D + action output

[ ] 背熟 4 个关键参数：
    - horizon=16：预测 16 步 action
    - n_obs_steps=2：用 2 帧历史观测
    - n_action_steps=8：执行前 8 步
    - num_inference_steps=10：DDIM 去噪 10 步

[ ] 分清 DDPM 和 DDIM：
    - DDPM：训练时预测 noise，推理时也按训练步数去噪，慢
    - DDIM：推理时可以用更少步数（如 10 步代替 1000 步），快

[ ] 理解 prediction_type=epsilon：
    模型预测的是加入的噪声，反推干净 action

[ ] 准备一句话总结：
    "Diffusion Policy 把 action 生成当成条件去噪过程，能表达多模态分布，
     但推理慢，且仍然依赖数据覆盖。"

面试话术示例：
    "Diffusion Policy 用 UNet 做条件去噪，以图像和状态为 condition，从纯噪
     声逐步生成 action chunk。相比 ACT，它通过 score matching 保留多模态分
     布，不会把多个正确轨迹平均掉。我把 num_inference_steps 从 10 降到 5，
     推理速度提升约一倍，动作质量没有明显下降。"

[ ] 背熟 3 个关键参数：

    - train_expert_only=true：只训练 action expert

[ ] 理解 VLA 的优势：

[ ] 准备一句话总结：

3.2 以 SmolVLA rename_map bug 为例
----------------------------------
[ ] 报错信息原文：
    ValueError: All image features are missing from the batch. At least one expected.

[ ] 假设树：
    1. 数据加载有问题？→ 检查 batch keys，有 observation.images.camera1/2/3
    2. 模型配置有问题？→ 检查 policy.config.image_features，还是 cam_high/...
    3. 预处理没生效？→ 检查 RenameObservationsProcessorStep，确实改了 batch key
    4. 配置和预处理不同步？→ 定位到 make_policy() 填充 input_features 时没应用 rename_map

[ ] 验证方法：
    打印 batch.keys() 和 policy.config.image_features.keys()，发现交集为空

[ ] 修复：
    在 make_policy() 中对 cfg.input_features 应用 rename_map

[ ] 反思：
    框架的「配置-数据-预处理」三层一致性容易出 bug，以后要先打印 key 做 sanity check

面试话术示例：
    "训练 SmolVLA 时第一个 step 就报 All image features are missing from
     the batch。我先打印了 batch 的 keys，发现是 camera1/2/3；又打印了
     policy.config.image_features 的 keys，发现还是 cam_high/...。这说明
     RenameObservationsProcessorStep 改了运行时 batch，但 make_policy()
     初始化配置时没同步改。于是我在 factory.py 里给 cfg.input_features
     也应用了 rename_map，问题就解决了。这个 bug 让我意识到，框架里配置、
     数据、预处理三层的一致性需要显式校验。"

7.1 Transformer 基础
--------------------
问题：ACT 和 SmolVLA 的 action expert 都用了 Transformer。请描述
self-attention 的计算过程，以及为什么要用 multi-head attention 而不是
single-head？Transformer 里的 position encoding 是干什么的？ACT 里输入的
image token 和 state token 需要加 position encoding 吗？为什么？

考察点：
- Q/K/V 的计算、softmax(QK^T/sqrt(d_k))V
- multi-head 让模型在不同子空间学习不同关系，增强表达能力
- position encoding 给序列注入位置信息，因为 attention 本身没有顺序概念
- ACT 的 image token 来自 CNN/ViT，通常需要加 position encoding；state token
  是低维向量，通常也加可学习的 position embedding

参考答案要点：
- self-attention：输入 X 分别乘 Wq/Wk/Wv 得到 Q/K/V，计算
  softmax(QK^T/sqrt(d_k))V 得到输出
- multi-head：把 d_model 拆成 h 个头，每个头独立做 attention，最后拼接
  再乘 Wo。好处是不同头可以关注不同模式（如全局依赖、局部关系）
- position encoding：因为 attention 是 permutation-invariant 的，必须显式
  告诉模型 token 的顺序。ACT 用 learnable position embedding 或 sinusoidal
  encoding


7.2 Diffusion 基础
------------------
问题：DDPM 的 forward process（加噪）和 reverse process（去噪）分别是什么？
noise schedule（如 squaredcos_cap_v2）的作用是什么？为什么不用线性 schedule？
训练时随机采样 timestep t，这个 t 的范围是多少？num_train_timesteps=100 和
num_inference_steps=10 是什么关系？

考察点：
- forward：x_t = sqrt(alpha_bar_t) * x_0 + sqrt(1-alpha_bar_t) * epsilon
- reverse：用神经网络预测 epsilon，逐步从 x_T 恢复到 x_0
- noise schedule 控制加噪速度，cosine schedule 在两端变化更平缓，训练更稳定
- t ∈ [1, T]，T = num_train_timesteps = 100
- num_inference_steps=10 是 DDIM 的推理步数，远小于训练步数，靠 DDIM 的
  非马尔可夫性质加速

参考答案要点：
- forward process：逐步给干净 action x_0 加高斯噪声，T 步后变成纯噪声 x_T
- reverse process：训练 UNet 预测加入的噪声 epsilon，推理时从 x_T 开始
  逐步去噪生成 action
- squaredcos_cap_v2：改进的 cosine schedule，避免两端 beta 变化过快，
  提升生成质量
- num_train_timesteps=100：训练时把 [0,1] 分成 100 个 timestep
- num_inference_steps=10：推理时用 DDIM 跳步采样，10 步就能生成，速度
  提升 10 倍，质量损失很小


7.3 归一化
----------
问题：BatchNorm、LayerNorm、GroupNorm 的区别是什么？为什么 Diffusion
Policy 里用 GroupNorm 而不是 BatchNorm？pre_norm 和 post_norm 的区别是什么？
对训练稳定性有什么影响？

考察点：
- BatchNorm：按 batch 统计均值方差，依赖 batch size，训练推理不一致
- LayerNorm：按特征维度归一化，不依赖 batch，适合 Transformer
- GroupNorm：把 channel 分组归一化，不依赖 batch size，适合 CNN 和小 batch
- Diffusion Policy 的 UNet 是 CNN，batch size 可能很小（视频数据显存限制），
  且训练和推理的 batch 分布差异大，所以用 GroupNorm
- pre_norm：norm 在 residual 之前，训练更稳定，适合深层网络
- post_norm：norm 在 residual 之后，原始 Transformer 结构，深层容易梯度消失

参考答案要点：
- BatchNorm 在 batch 维度归一化，batch size 小或分布不均时效果差；推理时
  用 running mean/var，训练和推理行为不一致
- GroupNorm 把 channel 分成 G 组，在每组内归一化，不依赖 batch size，
  适合 Diffusion Policy 这种小 batch CNN
- pre_norm（x + Attention(LN(x))）比 post_norm（LN(x + Attention(x))）
  梯度更稳定，现代 Transformer 普遍用 pre_norm


7.4 优化器与学习率
------------------
问题：Adam 的 beta1 和 beta2 分别控制什么？为什么 beta1 通常设 0.9 或 0.95，
beta2 设 0.999？warmup 的作用是什么？为什么前 500 步要从很小的学习率开始？
cosine annealing 和 step decay 相比有什么优势？

考察点：
- beta1：一阶矩估计（momentum）的衰减率，控制梯度方向的记忆
- beta2：二阶矩估计（梯度平方）的衰减率，控制自适应学习率
- beta1=0.95 比 0.9 更平滑，适合 Transformer 训练
- warmup：初期参数随机，梯度大，小学习率防止 loss 爆炸；同时让模型
  逐渐适应数据分布
- cosine annealing：学习率平滑下降到接近 0，比 step decay 更稳定，
  不容易卡在局部最优

参考答案要点：
- Adam：m = beta1*m + (1-beta1)*g；v = beta2*v + (1-beta2)*g^2；
  m_hat = m/(1-beta1^t)；v_hat = v/(1-beta2^t)；theta -= lr * m_hat/(sqrt(v_hat)+eps)
- warmup 500 步：从 0 线性增加到 peak lr，避免初期大梯度破坏预训练权重
- cosine schedule：lr = min_lr + 0.5*(max_lr-min_lr)*(1+cos(t/T*pi))，
  平滑衰减，适合长训练


7.5 损失函数与正则化
--------------------
问题：ACT 用 L1 loss，Diffusion Policy 用 MSE (L2) loss，为什么？能不能互换？
grad_clip_norm=10.0 是干什么的？为什么 Diffusion Policy 特别需要它？
weight_decay=1e-6 这么小，weight decay 和 dropout 都是正则化，它们的作用机制
有什么区别？

考察点：
- L1 loss 对异常值更鲁棒，输出更稀疏；L2 loss 对异常值敏感，梯度更平滑
- ACT 预测 action chunk，L1 能减少极端动作的影响
- Diffusion 预测 noise，MSE 是标准选择，与 score matching 理论一致
- 梯度裁剪：防止梯度爆炸，特别是 Diffusion 训练初期噪声大，梯度不稳定
- weight_decay：L2 正则化，约束权重幅度；dropout：随机丢弃神经元，
  防止 co-adaptation

参考答案要点：
- ACT 用 L1 因为 action 可能有异常值（如遥操作抖动），L1 更鲁棒
- Diffusion 用 MSE 因为预测的是高斯噪声，MSE 对应最大似然估计
- grad_clip_norm=10.0：把梯度 norm 裁剪到 10，防止 UNet 训练初期梯度爆炸
- weight_decay 是参数层面的正则，dropout 是激活层面的正则；两者可以互补


7.6 VLA / SmolVLA 相关
----------------------
问题：SmolVLA 基于 SmolVLM2，VLM 的 vision encoder 通常用什么结构？为什么
训练时选择 freeze_vision_encoder=true？SmolVLA 的 action expert 和 LLM 的
decoder 有什么区别？为什么 action expert 可以比 LLM 小很多？如果把 chunk_size
从 50 改成 10，会发生什么？对训练和推理各有什么影响？

考察点：
- VLM vision encoder：通常是 ViT（如 SigLIP、CLIP ViT-L/14）
- freeze vision encoder：利用预训练视觉表征，避免小数据微调破坏；减少
  可训练参数，降低显存和过拟合风险
- action expert：轻量 transformer，专门生成动作序列；不需要语言生成能力，
  所以可以小很多
- chunk_size=50 → 10：动作 horizon 变短，长期规划能力下降；训练时序列
  变短，显存减少，但可能需要更频繁的 replanning

参考答案要点：
- SmolVLM2 用 SigLIP ViT 作为 vision encoder，已经在大量图文对上预训练
- freeze 可以保留预训练视觉知识，只训练 100M 的 action expert，RTX 3090
  也能跑
- action expert 只做动作生成， vocab 是连续 action 而非离散 token，
  结构更简单，参数量可以小 5~10 倍
- chunk_size 减小：单次预测动作变少，长期任务（如 stack_bowls）可能失败；
  推理延迟降低，但 replanning 频率增加，可能更不稳定

9.1 第 1 轮：技术一面（VLA / 模仿学习算法工程师）
-------------------------------------------------

### 回答中的错误或不准确之处

1. ACT 的 loss 和 action chunk
   - 错误表述：L1 损失作用在单步 action 上，然后对 chunk 取平均
   - 正确理解：
     * 训练时：模型一次性预测 50 步 action chunk，loss 在整个 chunk
       的每一帧上计算 L1，再平均
     * 推理时：用 temporal ensemble，把当前 chunk 和历史 chunk 按
       权重平均，执行前 n_action_steps 步
   - 关键区别：训练时的 loss 计算 ≠ 推理时的 ensemble 机制

2. Diffusion Policy 的 num_inference_steps
   - 错误表述：num_inference_steps=10 代表加噪 10 次、去噪 5 次
   - 正确理解：
     * num_inference_steps 就是去噪（denoise）的步数
     * 训练时随机采样 timestep t，给 action 加噪，模型预测噪声 epsilon
     * 推理时从纯噪声开始，迭代 num_inference_steps 次逐步去噪
     * DDIM 的优势就是可以用更少步数达到相近效果

3. SmolVLA 失败原因判断
   - 现场说法：视觉定位精度不够，夹爪在碗上方但不够往下
   - 文档说法：夹爪闭合深度不够，夹得很浅，夹空气搬运空气
   - 问题：两个说法不一致，面试官会怀疑是否真正分析清楚
   - 改进：统一为"视觉定位大致正确，但夹爪闭合深度/物理接触精度不足"

4. 工程 bug 定位过程
   - 错误表述："报错后检查上下游数据是否统一，看了源码"
   - 问题：太笼统，没有具体 debug 路径
   - 改进：按"报错信息 → 假设树 → 验证方法 → 定位 → 修复 → 反思"
     的结构讲

5. 提高成功率方案
   - 错误表述：用强化学习导出更多训练数据
   - 问题：跳跃太大，不具体，缺乏优先级
   - 改进：按 P0（数据/步数）→ P1（多任务/增强）→ P2（RL/更强 VLA）
     的优先级回答

### 回答中的亮点

- 真机经验：20 次测试、16 次成功、偏移 5cm 掉到 10%，量化意识好
- 知道 ACT 用 L1 损失，Diffusion 用 epsilon 预测
- 主动分析评估布局和数据集分布
- 在真机 ACT 上看过 attention map
